¶ NPM Modules by Category

This page gives a short description of the used and preferred npm modules in different categories. It's neither meant to be a complete list or show the best modules but more my selection from the ever growing node modules list.

¶ General modules

¶ Development

  • alinex-builder helps administer common development tasks on the command line
  • mocha Simple, flexible, javascript test framework
  • chai assertion library that can be paired with mocha
  • alinex-worktime command line tool to log worktimes

¶ Language tools

  • CoffeeScript a language that compiles into JavaScript but is easier to read and write
  • alinex-async an utility module which provides straight-forward, powerful functions for working with asynchronous code (a superset of async)
  • Memoizee make a function or method using cache for results
  • Debug tiny node.js & browser debugging utility for your libraries and applications
  • carrier stream line reader

¶ Date and Time

  • Moment Parse, manipulate, and display dates and times
  • Chrono natural language date parser

¶ Data Storages

¶ Filesystem

  • alinex-fs enhancement for the node fs module (same as mkdirp, fs-extra, graceful-fs, ncp, rimraf, wrench and more)
  • chokidar fs-watch wrapper
  • minimatch glob like file name matching
  • POSIX => not used because not platform independent

¶ File Formats

  • js-yaml YAML parser (to read configurations)
  • xml2js Conversion from simple XML to JavaScript objects

¶ Databases

  • alinex-mysql easy to use mysql library (using mysql)

¶ Command line support

  • chalk easy way to get colorful output mainly on the command line.
  • yargs a light-weight, expressive, and powerful command-line framework
  • Prompt easy and beautiful command-line prompt

¶ Services

  • Request Simplified HTTP request client

¶ Server

¶ Management

  • Forever a simple CLI tool for ensuring that a given node script runs continuously

¶ Maintenance

  • Winston extensible multi-transport async logging library
